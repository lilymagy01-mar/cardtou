# 🌷 카드투유 (CardToYou) 메시지 카드 & 패널 출력 서비스 개발 계획서 및 태스크 리스트

## 1. 프로젝트 개요
본 프로젝트는 독자적으로도 실행 가능하지만, 궁극적으로 **SaaS형 구독 ERP 앱에 원활하게 이식(Integration/Embedding)**될 것을 염두에 두고 구조화하는 **'메시지 카드 자동 렌더링 및 출력 앱'**입니다. 별도의 프린트 브릿지 프로그램 없이 **일반 레이저 프린터**로 고품질 카드를 즉시 출력할 수 있도록 설계합니다. AI 이미지 생성 기술을 통해 세상에 하나뿐인 커스텀 카드를 제공합니다.

### 🎯 SaaS ERP 이식 및 무인 매장 확장을 위한 핵심 아키텍처
* **Multi-tenant(다중 입점사) 구조 대응:** 구독형 SaaS 특성을 고려해, 상점(Shop ID)별로 용지 여백, 보유 템플릿, 과금 내역이 완벽히 분리·관리되는 데이터베이스 설계 구조 채택.
* **Seamless API & Webhook 기반 데이터 수신:** ERP 본진(주문/고객 관리 시스템)에서 발송된 고객의 메시지 및 희망 사이즈를 API나 Webhook으로 지연 없이 즉시 수신하는 독립적인 백엔드 구성.
* **모듈형 UI (Micro-frontend):** ERP 관리자 페이지 내비게이션에 탭(iframe 또는 컴포넌트 형태)으로 위화감 없이 삽입될 수 있는 유연한 프론트엔드 설계.
* **🚀 무인 매장(Kiosk) 즉시 출력 지원 (추후 확장):** 오프라인 무인 매장 키오스크 환경에서 고객이 직접 메시지와 테마를 선택, 유료 결제 후 **현장에서 즉각(Real-time) 카드가 출력되도록 결제 모듈 및 로컬 프린터 직결 API** 세팅 고려.

## 2. 권장 인프라스트럭처 (Infrastructure Stack)
본 앱은 고해상도 이미지 처리(`canvas`)와 PDF 생성(`pdfkit`)이라는 무거운 렌더링 작업을 수행하므로, 일반적인 서버리스 환경보다는 **네이티브 라이브러리(Cairo, Pango 등)를 안정적으로 지원하는 컨테이너 환경**과 **다중 입점사(Multi-tenant) 처리에 특화된 DB**의 조합이 매우 중요합니다.

* **[Database & Storage & Auth] Supabase (강력 추천):**
  * **이유:** PostgreSQL 기반의 BaaS로, Row Level Security(RLS)를 통해 입점사(Tenant)별 데이터와 과금 내역을 완벽히 격리할 수 있습니다. 이미 제공되는 `Supabase Storage`를 활용해 기본 샘플 및 생성된 고해상도 PDF 파일을 즉시 저장 및 서빙하기에 가장 이상적입니다.
* **[Rendering Engine - Backend] Google Cloud Run 또는 Fly.io (Docker 컨테이너):**
  * **이유:** 서버리스 웹호스팅(Vercel 등) 환경은 50MB 용량 제한 및 OS 종속 라이브러리 설치 불가로 인하여 폰트 깨짐이나 Canvas 렌더링 에러가 발생하기 쉽습니다. 따라서 로직을 Docker로 감싸 자동 스케일링이 가능한 Google Cloud Run과 같은 컨테이너 플랫폼에 배치해야 가장 안정적인 렌더링과 트래픽 관리가 가능합니다.
* **[Client UI & Admin] Vercel (Next.js):**
  * **이유:** 관리자 UI와 무인 매장의 키오스크 프론트엔드 환경은 Next.js 기반으로 구축해 Vercel에 배포함으로써 전 세계 어디서든 가장 빠른 응답 속도를 제공합니다.

## 3. 주요 스펙 및 기능
* **출력 확장 (카드 & 패널 동시 지원):** 일반 소형 레이저 프린터(메시지 카드용)뿐만 아니라 대형 프린터/플로터(화환 패널, 홍보 패널 등)에 맞춘 큰 규격(A3, A2, A1 등)의 포스터/패널 출력용 고해상도 PDF 생성 모드.
* **템플릿 시스템:**
  * 기본 템플릿: 주제별(감사, 축하, 위로 등) 최소 10가지 기본 카드/패널 샘플 이미지 제공
  * AI 템플릿 모드: 선택된 테마의 AI 프롬프트를 기반으로(Antigravity API 연동) 신규 이미지 즉석 생성
* **간단한 캔버스 에디터 (Mini Canva 제공):**
  * 드래그 앤 드롭 텍스트 이동, 폰트 종류 변경, 크기 및 색상 조절 등 직관적 편집.
  * AI가 생성했거나 제공된 이미지 위에 스티커나 로고 등 간단한 이미지 오버레이 기능 지원 (템플릿 큐레이터 '샘플리' 담당).
* **렌더링 엔진:**
  * **반 접이식(Folded) 카드 대응:** 출력물 반으로 접었을 때 우측 하단/중앙에 위치하도록 데칼코마니 좌표 계산 (레이아웃 마스터 '각도기' 담당)
  * **텍스트 자동 포매팅:** 글자 수에 따른 폰트 Auto-scaling 및 Word-wrap 알고리즘 적용 (타이포 전문가 '폰트김' 담당)
  * **규격 대응:** A3, A4, A5, B4, B5 등 다양한 용지 크기별 사이즈 변환 및 Bleed/Margin 적용

## 4. 핵심 워크플로우 (이용 순서)
1. **주문 접수:** 고객이 메시지를 입력하고 테마(예: '생일', '감사') 설정.
2. **이미지 선택/생성:**
   * **[기본 모드]** 테마에 맵핑된 DB의 기본 샘플 이미지 호출
   * **[AI 모드]** 안티그래비티 API에 테마 프롬프트를 전송하여 1:1 맞춤형 이미지 렌더링
3. **렌더링 엔진 가동:**
   * 용지 규격과 마진에 맞춰 배경 이미지 리사이징/배치
   * 접는 선(Folding Line) 가이드라인 노출 (옵션)
   * 텍스트를 최적의 폰트 사이즈로 변환하여 지정된 좌표에 합성
4. **PDF 생성 및 출력:**
   * 300DPI급 Print-Ready PDF 생성 (프린트박 담당)
   * 연결된 일반 레이저 프린터로 전송

## 6. 단계별 개발 태스크 리스트 (Task List)

### Phase 1: 렌더링 코어 로직 & 수학적 계산 구축 (Architecture)
- [ ] 다양한 용지 규격(A4, A5, B5 등) 대응 DPI -> Pixel 변환 팩토리 함수 작성
- [ ] Folder-Card(반 접이식) 좌표 계산 로직 (중앙점 산출 및 메시지 위치 계산)
- [ ] 텍스트 처리 유틸리티: Width 측정 기반 Word-wrap (단어 단위 줄바꿈) 함수 개발
- [ ] 텍스트 Auto-scaling 엔진: 바운딩 박스 크기와 메시지 길이에 따른 최적 Font Size 계산 로직 구현

### Phase 2: 이미지 합성 & 템플릿 모듈 개발 (Design & Template)
- [ ] Node.js 환경에서의 이미지 합성 캔버스(Canvas API 또는 Sharp 라이브러리) 환경 세팅
- [ ] JSON 기반의 템플릿 스키마 설계 (배경 이미지 URL, 텍스트 시작 X/Y 좌표, 폰트/컬러 정보 포함)
- [ ] 기본 샘플 이미지 API 연동 및 Storage 저장소 설계
- [ ] AI 이미지 생성 파이프라인 (안티그래비티 API 연동 + 프롬프트 Injection 처리)

### Phase 3: High-Res PDF 변환 & 출력 모듈 개발 (Print & Output)
- [ ] 합성된 최종 캔버스 이미지를 300DPI Print-Ready PDF로 변환하는 로직 작성 (PDFKit 또는 jsPDF 사용)
- [ ] PDF 생성 시 Folding Line 추가 렌더링 옵션 개발
- [ ] PC 또는 Server에서 레이저 프린터로 자동 팝업/스풀 전송 기능 처리

### Phase 4: 단독 에디터 사용성 극대화 (Standalone Perfection)
"ERP 연동이나 자동화는 추후 단계로 미루고, 현재 앱만으로도 충분히 유용하고 강력한 디자인 에디터가 되도록 구축"
- [ ] 브라우저 로컬 스토리지 기반 디자인 저장/불러오기(Save & Load) 갤러리 기능
- [ ] 사용자 로컬 이미지 업로드 기능 (커스텀 배경 템플릿)
- [ ] 웹 폰트(Google Fonts 등) 연동을 통한 다양한 글꼴 선택 및 폰트 커스터마이징 기능
- [ ] 텍스트 블록 정렬(좌/우/가운데) 및 레이어 제어(Z-index) 기능
- [ ] 다양한 용지 규격(A4, A5, B4, 엽서 등) 프리셋에 맞춘 캔버스 자동 스케일링 고도화

### Phase 5: 알파 테스트 및 전문가 리뷰
- [ ] MOCK 데이터를 활용한 기본 A4, A5 테스트 출력 및 텍스트 짤림(Overflow) 현상 검증
- [ ] 접선 위치 정확도('각도기' 검증) 테스트
- [ ] AI 생성 속도 및 레이저 프린팅 스풀링 부하 테스트

### Phase 6: 무인 매장 키오스크 결제 & 즉시 출력 연동 (추후 확장)
- [ ] 키오스크용 고객 프론트엔드(Client UI) 기획 및 개발 (메시지 입력, 테마 선택)
- [ ] PG(결제) 모듈 연동 및 결제 완료 후 트리거되는 Webhook 개발
- [ ] 결제 승인 즉시 키오스크 로컬에 연결된 프린터로 Spooling 하는 다이렉트 출력 파이프라인 최적화
